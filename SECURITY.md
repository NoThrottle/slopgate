# Security Policy

## Reporting a vulnerability

Please report vulnerabilities privately to the maintainers before opening a public issue.

Include:

- A clear description of the issue
- Reproduction steps
- Potential impact
- Suggested mitigation if known

The maintainers will acknowledge reports as quickly as possible and coordinate a fix and disclosure timeline.

## Scope

This project targets safety-focused, behavior-preserving obfuscation. Issues that bypass documented safety guardrails or cause unexpected runtime behavior are in scope.
